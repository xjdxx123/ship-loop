# ship-loop v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the ship-loop Claude Code plugin: one `/ship` command turns a vague idea or PRD into a shipped, commercializable product via a human-rich design phase and an autonomous build loop.

**Architecture:** A plugin (commands + agents + skills + hooks + scripts). Deterministic core is a zero-dependency Node state engine (`ship-state.mjs`) that owns `feature_list.json` and the Stop-hook gate; all agent behavior lives in markdown prompt assets that reference the spec (`docs/superpowers/specs/2026-06-10-ship-loop-design.md`) as source of truth.

**Tech Stack:** Node 20+ ESM (no runtime deps), `node --test`, bash (macOS/Linux), Claude Code plugin conventions (plugin.json / commands / agents / skills / hooks), GitHub Actions CI.

**Source of truth:** every prompt-asset task below lists REQUIRED frontmatter + REQUIRED content invariants. Full prose is authored at execution time from the design spec §-references given per task. Code tasks contain complete code.

### Task 2: State engine — schema + ship-state.mjs (TDD)

**Files:**
- Create: `templates/feature_list.schema.json`, `scripts/ship-state.mjs`, `test/ship-state.test.mjs`

CLI surface (consumed by gate.sh, conductor skill, commands — names are LOAD-BEARING):

```
ship-state.mjs init --dir D --product NAME
ship-state.mjs validate --dir D
ship-state.mjs add --dir D            # one feature JSON on stdin → prints assigned id
ship-state.mjs next --dir D [--count N]   # JSON array; deps satisfied; hotfix<bug<feature, then priority, then id
ship-state.mjs set --dir D --id F-001 [--status S] [--passes true|false] [--note ...] [--bump-attempts]
ship-state.mjs stats --dir D          # {"total","passed","parked","pending","in_progress","open_bugs","done"}
ship-state.mjs gate --dir D           # exit 0 iff done (every feature passed|parked)
ship-state.mjs learn --dir D          # {"lesson","tags"} on stdin → learnings.json append (timestamped)
ship-state.mjs lessons --dir D [--grep WORD]
ship-state.mjs stop-hook              # reads Claude Stop-hook JSON on stdin (uses .cwd)
```

Feature object schema: `id` `^F-\d{3,}$`, `title` str, `type` feature|bug|hotfix|spec-conflict, `priority` 1|2|3, `depends_on` [ids existing], `verification` [str, ≥1], `passes` bool, `status` pending|in_progress|passed|parked|reset, `attempts` int≥0, `contract` str|null, `notes` str. File: `{version:1, product, features[]}` at `D/docs/ship-loop/feature_list.json`. Hand-rolled validator (no deps); every read+write revalidates (anti state-rot, spec §7).

stop-hook semantics (spec §7): no `D/docs/ship-loop/ACTIVE` → exit 0 silently. `PAUSED` marker → exit 0. `stats.done` → exit 0. Else anti-spin: sha256(feature_list) vs `.gate-spin` {hash,count}; same hash → count+1 else 1; count ≥3 → append escalation to `NEEDS_HUMAN.md`, exit 0 (allow stop). Otherwise stdout `{"decision":"block","reason":"ship-loop: <pending> features remain (<open_bugs> bugs). Re-enter the conductor round protocol (skills/conductor)."}` and exit 0.

- [ ] **Step 1: Write failing tests covering:** validate accepts/rejects (missing verification, bad type, dup id, unknown dep), add assigns F-001→F-002 + defaults, next ordering (hotfix first, then priority, dep-gated, excludes passed/parked/in_progress), set transitions + bump-attempts, stats/gate done logic (passed+parked=done), learn/lessons roundtrip, stop-hook ×4 (no ACTIVE silent-allow; ACTIVE+pending block JSON; 3rd same-hash allow+escalation file; done allow). Use `node:test`, tmpdirs, `child_process.execFileSync` for stop-hook.

### Task 6: Agents (4 files)

**Files:** Create `agents/initializer.md`, `agents/implementer.md`, `agents/evaluator.md`, `agents/triage.md`. Frontmatter: `name`, `description` (boring+specific), `tools` (initializer: Read,Glob,Grep,Write,WebSearch,WebFetch; implementer: full edit+Bash; evaluator: Read,Bash,Glob,Grep + browser MCP via ToolSearch; triage: Read,Bash,Glob,Grep,Write), `model: inherit`.

REQUIRED behavior invariants:
- initializer (spec §3): derive feature_list ONLY from frozen docs; granularity "one sub-agent session per feature"; every feature ≥1 executable verification step; pipe each via `ship-state.mjs add`; never invent scope; log bets to ASSUMPTIONS.md.
- implementer (spec §4): read contract first; inject TECH_SPEC + relevant `lessons --grep`; TDD (verification steps before code); work ONLY in assigned worktree; final message = JSON `{milestoneId, codeComplete, summary, filesTouched[], parked[]}`; four park categories (money/withheld-secret/irreversible/deadlock) → stub + park, never block; never grade own work.
- evaluator (spec §4): default stance REJECT; verdict JSON `{refuted, evidence, commandsRun[], outputExcerpt, rubric:{design,originality,craft,functionality}, newFindings[], recommendReset}`; MUST run commands itself — a verdict without `commandsRun` is invalid; UI features → Playwright/Chrome MCP walk + screenshot; rubric calibrated by quoting DESIGN_SPEC; newFindings feed `ship-state.mjs add`; recommendReset when no rubric progress across attempts ("cannot hill-climb").
- triage (spec §3 path 2, §4 step 8, §6): brownfield audit → CURRENT_STATE.md (runs the app; failure-to-run = finding #1); smoke classification regression|flake|env (flake → quarantine note, never code change).
