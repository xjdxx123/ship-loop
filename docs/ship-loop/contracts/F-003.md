## Edge cases owned
- `cost-history.json` missing, unreadable, malformed JSON, or not an array → consumer
  falls back (branch 4), never crashes the gate; producer never overwrites a malformed
  file — it leaves it and reports, so corrupted measurements are inspectable, not
  laundered.
- Junk records (missing fields; zero, negative, non-numeric, or non-finite
  `features_done`/`est_tokens_total`) → filtered by the usable rule; the rule's
  `features_done ≥ 1` floor makes divide-by-zero impossible. Filtering can legitimately
  drop the count below 2 → fallback branch.
- Exactly 1 usable record → still the fallback sentence: one run is an anecdote, not a
  range (the frozen feature title promises a range from history, so history needs ≥2).
- All-parked delivery (`done 0/Y`) → `features_done` 0 fails the usable rule; the run
  contributes no rate.
- `min(r) = max(r)` (identical or duplicated rates) → the range collapses to a point;
  print the single figure once instead of `X–X`.
- Charter without a `token_budget_day` row → omit the charter-days clause (F-002 already
  tolerates the absent row; the quote must too).
- Free-form run logs: this dogfood run's own `loop-run-log.md` lines carry neither
  `done X/Y` nor `est tokens` figures → the producer appends NOTHING, and the next
  intake honestly quotes the fallback instead of fabricated history. Conductor round
  step 8 pins the parseable format for every future run.
- `stats` exit code 1 before done → pinned in step 1: read stdout, ignore the exit code.
- Degraded autonomous mode (`--autonomous`) auto-freezes but still passes the gate step:
  the estimate block goes to the loud start log — it is presentation, not an approval,
  so degraded mode loses nothing.
- Retro re-run on the same delivery → a duplicate record is possible; the median is
  robust to one duplicate and the close says it best: an offer, not a meter. Documented,
  accepted.
