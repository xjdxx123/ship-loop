## Done means
- `scripts/notify.sh` exists, is executable, passes `bash -n`, and implements exactly
  this CLI: `notify.sh <title> <body>` or `notify.sh --test` (no other flags). Platform
  detection is a bash `case "$OSTYPE"` (no external binary needed to detect): `darwin*`
  with osascript present → `osascript -e 'display notification "<esc-body>" with title
  "<esc-title>"'`; `linux*` with notify-send present → `notify-send -- "$title" "$body"`;
  any other OS or missing binary → echo fallback `[ship-loop notify] <title>: <body>`
  printed to stderr via `printf` (dash-safe).
- Arg parsing is positional and exact: `--test` is recognized only as the sole argument
  (argc==1); argc==2 is always title+body, even when `$1` is literally `--test` or
  dash-leading; every other argc (zero, one non-`--test` arg, three+) prints the
  one-line usage to stderr and exits 0.
- Never fails the caller: every path exits 0 — including the usage path, missing
  osascript/notify-send/curl, a failing send (stderr suppressed, falls through to the
  echo fallback), and an unreachable webhook. No `set -e` abort path can leak a
  non-zero exit.
- Optional webhook is additive, after the desktop path: iff `SHIP_LOOP_WEBHOOK` is set
  non-empty and curl exists, POST exactly
  `{"title":"<t>","body":"<b>","source":"ship-loop","ts":"<UTC ISO-8601>"}` as a
  single line with `curl -sS -m 5 -X POST -H 'Content-Type: application/json'`,
  failure swallowed. The `ts` value is `date -u +%FT%TZ` output (relay.sh house style),
  not a free-form "ISO-8601". JSON escaping covers backslash, double quote, newline
  (`\n`), carriage return, tab; osascript escaping covers backslash + double quote with
  newlines flattened to spaces.
- `--test` exercises the full real send path (including webhook when env is set) with
  fixed strings title `ship-loop` / body `notify self-test`, prints EXACTLY one stdout
  line — the marker `notify.sh self-test: path=<osascript|notify-send|echo> webhook=<on|off>`
  — and exits 0. `path=` names the transport selected by platform detection (OS +
  binary present), independent of whether the send succeeded (a failed send still falls
  through to the stderr echo fallback); `webhook=on` iff `SHIP_LOOP_WEBHOOK` is
  non-empty and curl exists. With the env var unset, this darwin machine's stdout is
  byte-exactly `notify.sh self-test: path=osascript webhook=off`.
- notify.sh's header comment contains the trigger-point table naming all three
  triggers: (1) gate budget-pause — `scripts/ship-state.mjs stop-hook` via
  `hooks/gate.sh`, explicitly marked "wired by F-002, NOT here"; (2) conductor
  NEEDS_HUMAN append, every new row; (3) ship-deliver handover (step 5) — plus the
  DESIGN_SPEC body convention: name the item, the file, and the next action; never
  "something needs your attention".
- `skills/conductor/SKILL.md` Exit section: the "every 5 newly parked items" sentence

- `skills/ship-deliver/SKILL.md` section 5 gains the handover-notification sentence

## Edge cases owned
- Zero args, one arg other than `--test`, or three+ args → one-line usage on stderr,
  exit 0 (a mis-wired caller is visible in logs but never broken). Two args are always
  title+body — even a literal `--test` or dash-leading title is sent, never parsed as
  a flag.
- Title/body containing double quotes, backslashes, newlines, CR, tabs, or a leading
  dash → correctly escaped for osascript and JSON; `notify-send --` and `printf '%s'`
  prevent option injection; payload stays one line.
- Empty-string title or body (`""` counts as provided) → sent as-is; message quality is
  the caller's contract per the header convention.
- darwin without a GUI session (SSH): osascript fails → stderr suppressed, echo
  fallback fires, exit 0.
- linux without notify-send, or unknown `$OSTYPE` → echo fallback, exit 0.
- `SHIP_LOOP_WEBHOOK` set but curl missing → webhook silently skipped; desktop path
  unaffected.
- Webhook unreachable, refusing, or hanging → `-m 5` bounds the wait; failure swallowed;
  exit 0.
- Stripped environment PATH (cron/hook contexts) → `$OSTYPE` detection and `printf`
  fallback need no external binary; still exits 0.
