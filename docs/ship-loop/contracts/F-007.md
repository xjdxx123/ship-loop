# Contract F-007 — permissions preflight: freeze-gate ask + conductor entry probe
Status: AGREED

## Problem being fixed
The Stop-hook gate keeps the SESSION alive, but a tool-permission prompt stops the
individual TOOL CALL: under Claude Code's default permission mode every
non-allowlisted Bash/Edit/Write/Agent call pops a confirmation dialog, and an
unattended build stalls on the first one nobody is there to click — the most common
unattended failure, ahead of budget burn (which F-002 already gates). No skill can
read the session's permission mode programmatically, so the preflight is two
human-shaped, observable halves: (a) the FREEZE GATE message asks the human to confirm
auto-approval before "go" (pinned sentence in the quoted autonomy message), and (b) the
conductor entry checklist runs one harmless probing Bash call that WOULD prompt under
default mode — if it just required manual approval, the session is provably not armed,
and whoever approved the probe is by construction present to read the warning. If
nobody is present, the probe call itself hangs — the stall surfaces at second zero of
the build instead of round 3.

Claude Code permission reality, grounding the wording: modes are default (prompts),
`acceptEdits` (auto-approves FILE EDITS only — Bash still prompts), plan (mutating
tools do not run at all), and `bypassPermissions` / `--dangerously-skip-permissions`;
sessions may also carry a settings allowlist scoped per tool/pattern. Headless paths
are already armed — the flag appears at exactly these lines (cited verbatim):
- `scripts/headless.sh:21` — `( cd "$DIR" && claude --dangerously-skip-permissions -p "/ship:resume --autonomous" ) || true`
- `scripts/headless.sh:31` — `( cd "$DIR" && claude --dangerously-skip-permissions -p "/ship:resume --autonomous" ) || \`
- `scripts/relay.sh:44` — `claude --dangerously-skip-permissions -p "/ship:resume" >> "$SD/relay.log" 2>&1 || \`

## Out of scope
- Any engine change. No mode-detection code path: a skill cannot read the session's
  permission mode (no env var, no settings introspection contract), so the falsifiable
  surface is the pinned text, the cold probe drill, the validator, and the unchanged
  34/1 test suite — this is a prompt-asset feature.
- Writing settings/allowlists or auto-arming `bypassPermissions` on the human's
  behalf: arming auto-approval is exactly the consent the gate asks for; a preflight
  that grants it to itself answers its own question.
- notify.sh wiring or a NEEDS_HUMAN.md row for the entry warning (human present by
  construction — they just clicked the approval; F-004's row→notify invariant is for
  rows, and this writes none).
- Edit/Write-class probes. One harmless Bash probe is the contract; a session whose
  allowlist covers Bash but not Edit/Write passes the probe and can still prompt later
  — residual risk documented in edge cases, accepted (probing every tool class means
  multiple prompt burdens on every attended entry).
- `commands/ship-resume.md` edits: it re-enters `skills/conductor` after its own

## Edge cases owned
- **`acceptEdits` misread** (human hears "auto mode", arms shift+tab accept-edits):
  Bash still prompts → the probe call itself prompts at entry → warning fires. The
  gate sentence is the ask; the probe is the authority — whatever the human believed,
  the probe measures ground truth.
- **Probe denied** instead of approved: the prompt APPEARING is the observation; the
  command never ran, so there is no file to clean. Same branch: warn, continue.
- **Nobody present under default mode**: the probe call hangs on its own dialog — the
  stall surfaces at entry (second zero) instead of mid-round 3, which is this
  feature's floor: it cannot un-stall a prompting session, only move the stall to the
  gate where the go-granting human most likely still is.
- **"Always allow" clicked on an earlier entry-step Bash call** (step 1 `$SHIP
  validate` or step 2 ACTIVE touch under default mode): the probe then runs clean —
  correctly, Bash is now session-allowlisted; the remaining Edit/Write gap is the
  documented residual above.
- **Allowlist covers Bash but not Edit/Write/Agent**: probe passes, later edits may
  prompt — residual, documented; the gate sentence names the full set
  Bash/Edit/Write/Agent so a careful human arms all four.
- **Headless / relay legs**: every `claude` invocation carries
  `--dangerously-skip-permissions` (headless.sh:21, headless.sh:31, relay.sh:44) →
  probe runs silently, zero prompts, zero warning noise; degraded mode loses nothing.
- **Interactive `--autonomous` without the flag**: design-intake auto-freezes with no
  human at the gate, but conductor entry still probes; whoever launched the session
  interactively is present at entry to see the warning.
- **Plan mode**: mutating tools do not run at all — entry step 2 (ACTIVE touch)
  already cannot complete, so the run never reaches the probe pretending to be armed.
- **`touch` fails for filesystem reasons** (read-only checkout): a failed exit code is
  NOT the prompt signal — the probe observes whether approval was demanded, not
  whether the command succeeded; and a read-only `docs/ship-loop/` already broke
  step 2's ACTIVE touch before the probe.
- **Probe killed between `touch` and `rm`** (SIGKILL mid-call): a stray
  `docs/ship-loop/.preflight` dot-file may persist; harmless, untracked, and the next
  entry's probe re-creates and removes it.
- **`docs/ship-loop/` missing at step 3**: unreachable — entry step 1 verified the
  frozen docs live there (else: back to design-intake).
- **Renumber blast radius**: no landed verification command greps an entry-checklist
  digit (checked F-001..F-006, F-010, F-012); F-005's prose comment "entry checklist
  step 3" describes what is now step 4 — its five conductor grep pins all hold at
  landed counts (re-run above).
