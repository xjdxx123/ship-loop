# Contract F-002 — budget hard-stop in the Stop-hook gate
Status: AGREED

## Done means
- `scripts/ship-state.mjs stop-hook` gains a budget gate inserted after the existing ACTIVE / PAUSED / feature-list-valid / `stats.done` checks and BEFORE the spin-stall logic, with every pre-existing check untouched and in its current order. The gate evaluates only when ALL hold: hook stdin's `transcript_path` is a non-empty string; `<cwd>/docs/ship-loop/BUILD_CHARTER.md` is readable (cwd = `input.cwd || process.cwd()`, the hook's existing resolution); its `token_budget_day` row parses to a safe positive integer (parsing bullet below); and the F-001 cumulative `total` (input + output + cache_read + cache_creation) summed from the transcript is strictly greater than the budget — `total > budget`, exact equality is NOT a breach (TECH_SPEC: "cumulative total > charter token_budget_day"). When any precondition fails, hook behavior is byte-identical to today (fail-open: no enforcement, no new output, no new files, block reason string unchanged). The entire evaluation is additionally wrapped so an unexpected internal throw degrades to no-enforcement — the hook never crashes or alters anyone's session.
- The F-001 summation machinery is REUSED, not duplicated: the per-line usage-summing loop moves into one shared module-level helper called by both `cost` and the stop-hook, so each of the four usage field names (e.g. `cache_creation_input_tokens`) appears exactly once in `scripts/ship-state.mjs` (grep-pinned in verification). `cost`'s CLI surface stays byte-identical to contract F-001 — all five tests in `test/ship-state.cost.test.mjs` pass unmodified and the frozen command `node scripts/ship-state.mjs cost --dir . --transcript test/fixtures/transcript.jsonl` still prints the pinned total-1128 JSON with empty stderr. From the hook path the helper is silent: the stop-hook writes NOTHING to stderr in any case (the "cannot read transcript" warning remains a `cost`-CLI-only behavior); an unreadable transcript yields zeros = no breach. Transcript reading stays `readFileSync` at this single shared call site — streaming for >512MiB transcripts is F-010's, explicitly out of scope (boundary + justification in Out of scope).
- Charter parsing is pinned: budget comes solely from `docs/ship-loop/BUILD_CHARTER.md` under the product dir, via the first match of the tolerant row regex `/^[ \t]*\|[ \t]*token_budget_day[ \t]*\|[ \t]*([0-9][0-9_,]*)/m` over the raw file text — horizontal whitespace (`[ \t]`) ONLY, never `\s`, because `\s` matches `\n`: the `\s*` variant provably captures digits off the NEXT line when a malformed row ends at the key's closing pipe (`| token_budget_day |` followed by `3000 ...` yields budget 3000 from a line that is not the value cell). The match must be provably single-line. The captured digits have `,` and `_` stripped, then `Number(...)`; enforcement requires `Number.isSafeInteger(v) && v >= 1`. Tolerated and parsed: leading indentation, whitespace around cells, separator values (`1,500,000`, `1_500_000`), trailing text in the value cell (`2000000 (est)`). No enforcement and no crash for: charter file missing/unreadable, row absent, value cell not starting with a digit (`TBD`, `~2M`), value cell empty at end-of-line (no cross-line borrowing), value `0`, value beyond safe-integer range. Key match is exact and case-sensitive (`token_budget_day`, the frozen charter/template spelling); with multiple rows, the first MATCHING line wins — a non-digit-leading row is not a match, so scanning continues past it (a `TBD` row disables enforcement only when no later line in the file matches the regex).
- On breach the hook performs, in order, with one `const ts = new Date().toISOString()` shared by all artifacts and `total`/`budget` rendered as plain base-10 integers (no separators):
  1. append to `<cwd>/docs/ship-loop/NEEDS_HUMAN.md` exactly
     ``- [ ] ${ts} gate: token budget exceeded (${total} tokens > token_budget_day ${budget}) — run paused; review spend, then raise token_budget_day in docs/ship-loop/BUILD_CHARTER.md and rm docs/ship-loop/PAUSED to resume\n``
     (house shape of the existing stall row; names the item, the file, and the next action);
  2. write `<cwd>/docs/ship-loop/PAUSED` with content ``token budget exceeded: ${total} > ${budget} at ${ts}\n`` — its existence is the pause, and `relay.sh` already disarms on PAUSED (relay.sh:39), so necromancy disarms with zero extra wiring;
  3. best-effort `unlinkSync` of `.gate-spin` (a later resume must not inherit a phantom stall count);
  4. fire the notify call (next bullet);
  5. exit 0 with EMPTY stdout — allow the stop.
  Each write step is individually best-effort (try/caught): once breach is decided the hook ALWAYS allows the stop with exit 0 even if a write fails — stopping is the safe direction when over budget. While PAUSED exists, later stops exit at the existing PAUSED check before the budget gate, so a paused run never accumulates duplicate rows or notifications.
- The breach fires exactly one notification: `spawnSync('bash', [notifyPath, title, body], { stdio: 'ignore', timeout: 8000 })` inside try/catch, guarded by `existsSync(notifyPath)`. The timeout is 8000ms, not 10000: the rule is "strictly above notify.sh's own worst legitimate bounded path, no more" — notify.sh's only long pole is the webhook `curl -m 5` (5s hard cap) after the desktop transport, so 8000 = 5000 + 3000 transport/exec slack; a wedged osascript then holds the user's Stop for at most 8s instead of 10. `stdio: 'ignore'` is load-bearing beyond tidiness: notify.sh's transport-less fallback echoes `[ship-loop notify] ...` to ITS stderr, and ignoring child stdio is what keeps the hook's own stderr empty on GUI-less CI. The pinned value is grep-verified (verification 3). `notifyPath = join(dirname(fileURLToPath(import.meta.url)), 'notify.sh')` — notify.sh resolved as a sibling of the running ship-state.mjs, correct in both plugin-install and repo-checkout layouts, and `bash` as argv0 removes any exec-bit dependency. Pinned strings per the conductor convention (item, file, next action — never "needs attention"): title `ship-loop: gate paused (budget)`; body ``NEEDS_HUMAN.md: token budget exceeded (${total} > ${budget} tokens) — raise token_budget_day in docs/ship-loop/BUILD_CHARTER.md, then rm docs/ship-loop/PAUSED``. A missing, broken, or hanging notify.sh never changes the hook's exit code, stdout, or the pause artifacts (notify runs after PAUSED is durable). `scripts/notify.sh` itself is NOT edited — its header's "wired by F-002, NOT here" stays accurate: the call site lives in ship-state.mjs.

## Out of scope
- **Large-transcript handling (>512MiB / V8 string limit) — F-010 keeps it.** `readFileSync` is retained at the single shared read site. Justification: F-010 has its own frozen verification (streaming-parse test + byte-exact cost output) and a streaming rewrite would churn the F-001-pinned `cost` path inside an enforcement feature; F-002 instead concentrates every transcript read into the one shared helper, so F-010's later one-site `node:readline` fix is inherited by both `cost` and this budget gate automatically. Until F-010 lands, a >512MiB transcript reads as zeros = no enforcement (fail-open; documented edge case, not silently rounded up).
- Cross-session / true per-day aggregation: the gate compares THIS session's transcript total (the only spend visible to the Stop hook) against `token_budget_day`; multi-session daily accounting remains the conductor's round-accounting discipline (charter row note, loop-run-log). No spend-ledger files are introduced.
- Dollar/price conversion and pricing tables (F-001 declared; unchanged — tokens only).
- Dedup of repeated `message.id` usage entries (F-001 declared; overcounting stays the conservative direction for a tripwire).
- Pre-freeze cost estimation (F-003); notify transport changes (F-004, landed); README / /ship:status surfacing of cost and pauses (F-009).
- Any edit to `skills/conductor/SKILL.md` — its Exit section already attributes the unattended budget notification to the stop-hook (F-002), and its "Budget exceeded → checkpoint, write NEEDS_HUMAN note, pause" round-accounting is conductor-side prose, not this code path.
- Soft warning thresholds (e.g. notify at 80%) — single hard threshold only.
- Budget keys other than `token_budget_day` (no per-feature/per-round budgets) and any new env-var override of budget or notify path.
- Resolving a relative `transcript_path` against `input.cwd` — the value is used as-is (Claude Code sends absolute paths; an unresolvable one reads as zeros, fail-open).

## Edge cases owned
- `transcript_path` absent, empty string, or non-string → no enforcement; behavior byte-identical to today.
- Transcript missing / unreadable / EISDIR → shared helper yields zeros → never a breach; no stderr from the hook path (the warning stays `cost`-CLI-only).
- Empty, blank-only, or malformed-line transcripts → zeros / partial sums per F-001's tolerant line rules; under-budget result, normal block.
- Charter file missing or unreadable; row absent; value cell not digit-leading (`TBD`, `~2M`); value `0`; value beyond `Number.MAX_SAFE_INTEGER` → no enforcement, no crash.
- Multiple `token_budget_day` rows → first MATCHING line wins; a non-digit-leading row is not a match, so scanning continues past it (`TBD` disables enforcement only when nothing later in the file matches). Separator values `1,500,000` / `1_500_000` → parsed correctly.
- Malformed row ending at the key's closing pipe (empty value cell at end-of-line) → no match, no enforcement: the `[ \t]`-only regex cannot borrow digits from the next line (test d pins this).
- `total` exactly equal to budget → NOT a breach (strict `>`, per TECH_SPEC wording).
- Breach conditions met but run is done, PAUSED already exists, ACTIVE is absent, or feature list is missing/invalid → the existing earlier checks win; zero budget artifacts (gate position pins this).
- NEEDS_HUMAN.md does not exist yet at breach → `appendFileSync` creates it (the state dir provably exists — `feature_list.json` was just read from it).
- Human removes PAUSED without raising the budget → next stop legitimately re-pauses: one new row + one new notification per pause event; no spam while paused (PAUSED short-circuits first).
- notify.sh missing, non-executable, failing, or hanging → `existsSync` guard + try/catch + 8s `spawnSync` timeout (5s webhook `curl -m 5` ceiling + 3s slack) + ignored stdio; pause artifacts and exit code unaffected (notify fires after PAUSED is durable). Ignored stdio also swallows notify.sh's stderr-echo fallback on transport-less machines.
- fs write failure mid-breach (e.g. read-only state dir) → remaining steps still attempted, stop still allowed with exit 0 — stopping is the safe direction once over budget; the next session stop re-attempts if PAUSED could not be written (test h pins this).
- Stale `.gate-spin` from before the pause → unlinked at breach, so a resumed run cannot inherit a phantom stall count toward the 3-strike escalation.
- Malformed hook stdin JSON → exit 0 silently (existing rule, unchanged).

## Evaluator response
AGREED after one binding round. The proposal was unusually thorough; the audit therefore attacked its strongest claims empirically rather than re-deriving them. Findings:

**Held under attack (verified, not trusted):**
- Regex vs both real charters, run in node against the actual files: dogfood `BUILD_CHARTER.md` captures `1500000`, template captures `2000000`. The dogfood NOTES cell's `F-001/F-002` digits are provably inert — the capture group is anchored immediately inside the value cell and terminates at the first non-`[0-9_,]` character (the space before the third pipe), so NOTES digits are unreachable; first-match-wins is safe for both frozen shapes.
- Test (g) is present as claimed and pins both gate-position cases (run done, ACTIVE absent) — probe (4) of this audit confirmed it listed, not just implied.
- `relay.sh:39` claim verified against source: `[ -f "$SD/PAUSED" ] && { "$SELF" --disarm "$DIR"; exit 0; }` — necromancy genuinely disarms on PAUSED with zero new wiring.
- `hooks/gate.sh` is a pure `exec node ... stop-hook` shim; stdin forwarding claim accurate. Both pre-existing suites send no `transcript_path`, so "they now also pin the no-enforcement path" is accurate as written.
- Verification demo 4's stub `notify.sh` is never chmod'd — the demo therefore actually proves the `bash`-as-argv0 no-exec-bit claim, not merely states it.
- Strict `>` matches TECH_SPEC line 22 ("cumulative total > charter `token_budget_day`"); equality-is-not-breach is pinned by test (c).

**Defects found and tightened in place:**
1. **Cross-line regex capture (real bug in the proposed pin).** JS `\s` matches `\n`. Reproduced: `"| token_budget_day |\n3000 tokens is the note"` → the proposed `\s*` regex captures `3000` from the NEXT line — a truncated row silently enforces a budget no row states. Regex re-pinned to horizontal whitespace only (`[ \t]`), verified to preserve every tolerated shape (both charters, separators, trailing text) while refusing the cross-line capture; new matrix item in test (d) pins it.
2. **First-match semantics were under-specified.** A `TBD` row is not a match, so scanning continues — a later digit-leading `token_budget_day` line anywhere in the file would win. Behavior kept (charters are frozen single-table docs) but the contract now states it precisely instead of implying `TBD` always means no enforcement.
3. **Notify timeout 10000ms → 8000ms, with a rule.** A Stop hook must not hold the user's session for longer than notify.sh's own worst legitimate bounded path: webhook `curl -m 5` (5s) + 3s desktop/exec slack = 8s. 10s was 2s of pure wedged-osascript exposure. Value now grep-pinned in verification 3. Also recorded that `stdio: 'ignore'` is load-bearing: notify.sh's transport-less fallback echoes to stderr, and ignoring child stdio is what keeps the hook's "stderr empty in all cases" true on GUI-less CI.
4. **Hermeticity only held for passing runs.** "Non-breach tests may use the repo engine" left a hole: a gate-ordering regression would make a "non-breach" case breach against the REAL notify.sh — a desktop banner during `node --test`, in exactly the failing state tests exist to catch. Rule tightened: every test in the budget file runs the copied engine + stub sibling; the repo engine is confined to the untouched suites, which never supply a `transcript_path` and cannot reach notify even under a buggy gate. (Demos 6–7 still use the repo engine, acceptably: they run only after the suite is green and the hermetic claim is scoped to `node --test`.)
5. **Claimed-but-unverified sweep (probe 5):** three Done-means claims had no matching verification command — (i) "each of the four usage field names appears exactly once" was grep-pinned for only one of the four → verification 3 now `grep -oE | sort | uniq -c` over all four; (ii) the spawnSync timeout value → `grep -c 'timeout: 8000'` added; (iii) "doc comment updated to name the budget gate" → pinned to the literal phrase `budget gate`, file-wide count exactly 2 (cost comment already has one). Additionally the write-failure exit-0 invariant ("stopping is the safe direction") was an owned edge case with no test → new test (h): read-only state dir, breach → exit 0, silent, notify stub still logs, root-skip guarded.

No fundamental mis-scope found: gate position, fail-open posture, reuse-not-duplicate refactor, frozen F-001 surface, and the F-010 streaming boundary are all correctly drawn and verifiable as amended.
