# Contract F-015 — docs polish: permission-stall failure row + F-008 measured-counts correction note
Status: PROPOSED

## Problem being fixed
Two documentation drifts, one feature. (1) `docs/failure-modes.md` catalogs 14 ways
loops die but omits the one `skills/design-intake` names "the most common unattended
failure, ahead of budget burn": the loop stalls mid-build on a tool-permission dialog
nobody will click. The Stop-hook gate keeps the SESSION alive but a permission prompt
stops the individual TOOL CALL — session alive, zero progress. The mitigation already
shipped as F-007 (freeze-gate ask + observable conductor entry probe + pre-armed
headless legs); the catalog just never gained the row. (2) AGREED contract F-008's Dial
paragraph records two baseline diagnostic counts (84 under full `--strict`, 75 reproduced
red under `--noImplicitAny`) without recording the tsc version that produced them.
Re-measured on the same JSDoc-free baseline (`886216b~1`), typescript 5.9.3 with
@types/node 22.19.20 reports 73 and 81 — the pinned invocation pins majors only, so
`npx -y` floats the minor/patch and bare counts are machine- and date-variant.
Contracts are append-only once AGREED, so F-008's prose stays byte-untouched and the
correction is an appended note, in the F-013 house format.
