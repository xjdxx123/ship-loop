# Contract F-013 — false mechanical claim in pinned prose: stats/gate exit-code attribution
Status: PROPOSED

## Problem being fixed
`skills/design-intake/SKILL.md` Cost estimate step 1 ends with a parenthetical claiming
`stats` "exits 1 until the run is done; read stdout, ignore the exit code". That is
false, and was false when written: from the first engine commit (61299af),
`stats({ dir })` in `scripts/ship-state.mjs` prints the rollup and carries no exit-code
logic at all — it exits non-zero only through the shared `fail()` path when the feature
list is missing or invalid. The exit-1-until-done behavior belongs to `gate`
(`process.exit(stats.done ? 0 : 1)`). Live proof on this very repo mid-run:
`stats --dir "$PWD"` exits 0 while `gate --dir "$PWD"` exits 1. The "ignore the exit
code" advice is also harmful as written — for `stats`, a non-zero exit means a
missing/invalid list, exactly the failure an agent must not ignore. AGREED contract
F-003 carries the same false attribution in three places (Done-means step 1's "Pinned
operational note" with its mechanism claim, the pinned-edits block parenthetical, and an
Edge-cases bullet). Contracts are append-only once AGREED (skills/contract-negotiation
rule), so F-003's existing text stays byte-untouched and the correction is an appended
note.
