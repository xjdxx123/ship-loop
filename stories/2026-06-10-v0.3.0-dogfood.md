# Story: ship-loop built v0.3.0 with itself

**What was attempted**: the full loop protocol, on the loop's own repo — Phase 0 frozen
docs (Round 2 amendments), 9 planned features (developer trust pack: cost metering,
budget hard-stop, notifications, PR mode, rollback, permissions preflight, cost
estimate, types-without-TS, release), FREEZE GATE with human "go", then 8 autonomous
conductor rounds of contract-negotiated implementer/evaluator pairs in isolated
worktrees.

**Outcome**: shipped. 16/16 features passed (9 planned + **7 discovered mid-run by
evaluators** — a 78% living-list expansion), gate exit 0, v0.3.0 tagged, CI green,
installed plugin updated 0.2.1 → 0.3.0. Zero features parked, zero K-breaker trips,
zero human interventions between "go" and delivery.

**Token cost**: ~3,000,000 subagent tokens (~35 agent dispatches across contract
proposals, negotiations, implementations, verdicts, retro). **This EXCEEDED the
charter's 1,500,000 budget by 2x.**

**Root cause of the overrun**: two compounding factors, both recorded by the
retrospective. (1) The budget gate (F-002) was built DURING this run and could not arm
for it — plugin hooks load at session start, and the conductor's own free-form
accounting drifted from its pinned format. (2) The living feature list grew 78% over
the frozen plan — discovery is the loop's best feature and its main cost multiplier;
the pre-freeze estimate had no discovery multiplier (it didn't exist yet either: F-003
was built this run too).

**What the mechanisms caught** (evidence the protocol pays for itself):
- Contract negotiation caught a real cross-line regex bug (`\s` matches `\n`) BEFORE
  any code was written (F-002).
- Evaluators discovered: the >512MiB fail-open that would have blinded the budget gate
  on exactly the most expensive sessions (F-010); the pr-mode × depends_on workspace
  gap (F-011); a crash-class CLI defect (F-014); two false mechanical claims in pinned
  prose (F-013, F-015); a one-token CWD-drift hazard (F-016).
- An evaluator refused to round up a defective contract verification command and ruled
  a justified equivalent instead (F-004's stripped-PATH trap).

**Remediation** (tracked in docs/ship-loop/protocol-notes.md, 6 notes): conductor
self-metering via `cost --transcript` so the gate's arming lag can't blind a run;
engine-emitted accounting lines (free-form drift broke the cost-history parse);
discovery multiplier in pre-freeze estimates; evaluator contract-flips must be
committed in-worktree; pin the correction-note format (independently reinvented twice).

**Untested paths, honestly**: REJECT → repair → K=3 → park/reset never fired in
production (16/16 first-attempt passes — thorough contracts, or an unexercised
circuit breaker; the fixture tests cover it, production has not). Panel mode (3-vote)
ran only in fixture form. The first real external-product run should watch both.
