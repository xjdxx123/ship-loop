# Stories

Run records for ship-loop itself and for products built with it. Honest ones.

Every story MUST include:
- **What was attempted** (the goal / PRD one-liner)
- **Outcome** (shipped / parked / killed — with the live URL or the wreckage)
- **Token cost** (estimate is fine; "unknown" is not)
- **Root cause** of anything that went wrong
- **Remediation** (what changed in the charter, the specs, or this repo)

Failures are more valuable than wins here. A story that says "the evaluator passed a
broken checkout because the contract's verification was vague" makes every future
contract sharper. (Format convention borrowed from cobusgreyling/loop-engineering.)
